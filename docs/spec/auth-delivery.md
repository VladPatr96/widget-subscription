# Спека на передачу — авторизация, доставка и иконка виджета

Согласованный, готовый к передаче артефакт карты
[#13](https://github.com/VladPatr96/widget-subscription/issues/13) (wayfinder).
Сводит шесть закрытых решений (#14–#19) в единый документ. **Ничего нового здесь
не решается** — это схождение уже принятого; деталь каждого решения живёт в своём
тикете и ассете, сюда вынесены суть и связки.

Надстраивается над **спекой MVP** (`docs/spec/mvp.md`, карта #1): тот эффорт
зафиксировал источник данных, стек, провайдерный порт, модель обновления и
визуальный язык на **borrow-only** сценарии (виджет читает токен Claude Code
read-only). Этот эффорт добавляет: **перенос на чистую машину**, где Claude Code
может быть не установлен, — а значит собственный вход виджета, безопасное хранение
его токена, инсталлятор и иконку приложения.

Статус: назначение карты #13. После согласования этого документа открытых решений
на карте нет.

Глоссарий домена — `CONTEXT.md`; действующая MVP-спека — `docs/spec/mvp.md`.

## 1. Модель учётных данных: два режима с фолбэком (тикет #17)

Деталь:
[Решение: модель учётных данных и выбор режима](https://github.com/VladPatr96/widget-subscription/issues/17).

Виджет работает в одном из двух режимов получения токена для
`GET api.anthropic.com/api/oauth/usage`:

- **borrow** — заимствует токен установленного Claude Code из
  `~/.claude/.credentials.json`, строго **read-only** (как в MVP §3, §6). Виджет
  этот файл **никогда не пишет и не рефрешит** (ротация refresh с reuse-detection
  отозвала бы семью токенов CC — разведка #14 §6).
- **own-login** — собственный OAuth-грант виджета (свой файл, свой рефреш). Для
  машин без Claude Code.

### Политика выбора — авто с override (вариант C)

- **По умолчанию авто**: валидный файл CC есть → **borrow**; нет → предлагаем
  **own-login**.
- **Override**: пункт-тумблер в меню трея «Источник входа: Claude Code ↔
  собственный вход» форсирует own-login даже при наличии CC. Выбор
  **персистентный** (прилипает до следующего переключения).
- **Оба доступны**: уважаем явный сохранённый выбор; при его отсутствии — приоритет
  **borrow** (read-only, не создаёт грант, безопаснее).

### Критерий «валидного» файла CC — три состояния borrow (вариант A)

1. **нет файла / не парсится / нет токена** → CC «нет» → предлагаем own-login;
2. **файл есть, токен живой** → borrow штатно;
3. **файл есть, токен протух** → borrow **сохраняется**, но состояние
   **деградировано**: серый донат + «Токен Claude Code истёк — откройте Claude
   Code, чтобы обновить» + подсказка про override. Протухший токен = **деградация,
   не форс own-login**: состояние временное, самовосстановимое (CC запустится —
   обновит).

### UX (жаргон режимов скрыт, вариант A)

- Первичный вход в own-login — **пустое состояние панели**: пояснение + кнопка
  **«Войти через Anthropic»**; трей-иконка нейтральная (серая, без дуги).
- Дубль пункта **«Войти…»** в меню трея (рядом с тумблером «Источник входа»).
- Пользователю не показываем слова «borrow/own-login». Формулировки по состоянию;
  тумблер подписан нейтрально. Одна строка при own-login: «Отдельный вход, не
  влияет на Claude Code».

## 2. Разведка OAuth-флоу Claude Code — факты (тикет #14)

Деталь:
[Разведка: OAuth-флоу Claude Code](https://github.com/VladPatr96/widget-subscription/issues/14),
ассет `docs/research/ticket-14-oauth-flow.md`.

Claude Code — публичный OAuth 2.0-клиент (Authorization Code + **PKCE**, без
client_secret). Виджет воспроизводит флоу и получает **собственный грант** (свою
семью refresh-токенов), не пересекающийся с грантом CC, **при условии** что хранит
токены отдельно, не перезаписывая файл CC.

- **client_id** (публичный, общий): `9d1c250a-e61b-44d9-88ed-5944d1962f5e`; PKCE
  метод `S256`.
- **Authorize**: `https://claude.ai/oauth/authorize` — вход по подписке Pro/Max
  (именно этот грант несёт лимиты подписки для `api/oauth/usage`).
- **Token**: `POST https://console.anthropic.com/v1/oauth/token`
  (**уточнение** разведки — в тумане MVP ошибочно значился `platform.claude.com`).
- **Scope**: этот эффорт запрашивает **`user:inference user:profile`** (см. §3.6;
  `org:create_api_key` из полного набора CC выкинут — least-privilege).
- **Redirect**: (a) loopback на localhost (RFC 8252, эфемерный порт) — основной для
  GUI; (b) hosted callback + ручная вставка кода — фолбэк.
- **Токены**: access `sk-ant-oat01-` живёт ~8 ч (`expires_in: 28800`); refresh
  `sk-ant-ort01-` **ротируется** — успешный рефреш возвращает новый refresh, старый
  инвалидируется; повторное использование = reuse-detection → отзыв всей семьи.
- **Разделение грантов подтверждено**: own-прогон PKCE — самостоятельная семья;
  reuse-detection на уровне семьи; держится, только если own-login пишет **свой**
  файл (§4 ниже), а не файл CC.

## 3. Флоу собственного логина (own-login) (тикет #18)

Деталь:
[Решение: как виджет логинится](https://github.com/VladPatr96/widget-subscription/issues/18).
Наследует seam `IWidgetLogin` из #17 и endpoints/PKCE/client_id из #14.

### 3.1 Получение кода — loopback основной + paste-фолбэк (вариант A)

- **Основной — loopback** (RFC 8252): виджет поднимает временный HTTP-listener на
  свободном эфемерном порту localhost, открывает браузер на
  `claude.ai/oauth/authorize`, ловит `code` в редиректе автоматически, гасит
  listener, меняет код на токены (`POST console.anthropic.com/v1/oauth/token`).
- **Фолбэк — hosted-paste**: `redirect_uri=console.anthropic.com/oauth/code/callback`
  + `code=true`; пользователь копирует показанный код в виджет. Та же PKCE-пара,
  другой `redirect_uri`.

### 3.2 Страница логина — внешний системный браузер (вариант A)

Открываем URL во внешнем браузере по умолчанию (`ShellExecute`/`Process.Start`),
**не** встроенный WebView. Канонический native-app OAuth (как сам `claude login`);
переиспользуются куки claude.ai; ноль лишних зависимостей — согласуется с
self-contained single-file доставкой (§5, никакого WebView2-рантайма).

### 3.3 UI во время/после логина — модальное окно-ожидание (вариант A)

- Старт из §1: пустое состояние панели + дубль в меню трея.
- После клика — небольшое **модальное окно** «Ожидаем подтверждения в браузере…»:
  спиннер, кнопка **«Отмена»**, ссылка **«Ввести код вручную»** (paste-фолбэк).
  Окно **переживает потерю фокуса** (трей-поповер закрывается при уходе фокуса в
  браузер — поэтому статус в отдельном окне, не в панели).
- После успеха: браузер показывает заглушку «Можно вернуться в виджет», listener
  ловит код, окно закрывается, панель перерисовывается с лимитами.

### 3.4 Ошибки / отмена / таймаут

- **Порт занят / listener не поднялся** → сразу paste-фолбэк (другой порт не
  пробуем).
- **Браузер не достучался (WSL/прокси), код не пришёл** → по таймауту предлагаем
  «ввести код вручную» + «повторить».
- **Пользователь закрыл браузер / «Отмена»** → гасим listener, закрываем окно,
  **тихий возврат** в пустое состояние (не ошибка).
- **`state` не совпал** → отвергаем код: «Не удалось подтвердить вход, попробуйте
  снова».
- **Обмен кода на токен провалился (сеть/4xx)** → сообщение + «Повторить»;
  verifier/state прогона **не** переиспользуем — новый вход = новый PKCE-прогон.
- **Таймаут ожидания кода = 3 минуты**, затем «вход не завершён» + «Повторить» /
  «Ввести код вручную».

### 3.5 Повторный вход и разлогин

- **Провал рефреша own-токена** (reuse-detection / протух refresh / отзыв) →
  **деградация**: серый нейтральный донат + «Сессия истекла — войдите снова» +
  кнопка «Войти через Anthropic». Чинит **сам виджет** повторным входом. Битые
  own-токены **очищаем** (не держим мёртвый refresh).
- **Явный «Выход»** — отдельный пункт меню трея, только в own-login: удаляет
  собственные креды → пустое состояние. **Отделён** от override-тумблера
  «Источник входа» (§1): тумблер переключает *приоритет источника*, «Выход»
  *сбрасывает own-грант*. (borrow «выхода» не имеет — файл чужой.)
- **Повторный вход = полный флоу §3.1–3.4** (новый PKCE, новый грант); тихого
  переиспользования старого кода нет.

### 3.6 Scope — `user:inference user:profile` (вариант B)

**`org:create_api_key` выкинут** — привилегия минтить API-ключи read-only виджету
лимитов не нужна никогда (least-privilege). `user:inference` — минимум для
`GET api/oauth/usage`; `user:profile` — ради осмысленного «Вход выполнен: <email>»
(own-грант — отдельный аккаунт, показать полезно).

## 4. Хранение и рефреш собственного токена (тикет #19)

Деталь:
[Решение: хранение и рефреш собственного токена](https://github.com/VladPatr96/widget-subscription/issues/19).

### 4.1 Хранилище и формат

Файл `%LOCALAPPDATA%\WidgetSubscription\credentials.json` (Local, **не** Roaming —
согласовано с доставкой §5). Отдельный ключ, чтобы файл был однозначно **не** файлом
Claude Code:

```json
{ "widgetOauth": { "accessToken": "…", "refreshToken": "…", "expiresAt": 0, "refreshTokenExpiresAt": 0, "scope": "user:inference user:profile" } }
```

Файл `~/.claude/.credentials.json` в own-режиме виджет **никогда** не пишет.

### 4.2 Шифрование в покое — plaintext (паритет с CC)

**Plaintext JSON**, защита — ACL пользовательской папки. Принятый компромисс:
долгоживущий refresh лежит читаемым (та же экспозиция, что уже принял CC). Побочно
упрощает будущий кросс-платформенный шов (нет DPAPI-привязки к Windows).

### 4.3 Триггер и каденс рефреша — ленивый just-in-time

Без отдельного таймера: перед выдачей токена own-source проверяет
`now + skew >= expiresAt` → рефрешит и отдаёт новый, иначе кэш. **`skew = 5 мин`**.
Привязан к существующему poll-циклу MVP (60 с + форс-при-открытии) — новых каденций
нет. При 401 от usage — одна попытка рефреш+ретрай (own-грант рефреш разрешён,
в отличие от borrow).

### 4.4 Ротация refresh безопасно

- **Single-flight** (`SemaphoreSlim`) вокруг рефреша в own-source: poll и
  форс-при-открытии никогда не рефрешат одним токеном параллельно (иначе
  reuse-detection отзовёт own-семью). Дополняет engine-level `SemaphoreSlim` MVP.
- **Атомарная запись всего файла**: temp-файл + `File.Move`/replace — access +
  refresh + expiresAt ложатся вместе или никак; новый триплет **персистится до**
  использования нового access (обрыв записи не должен терять ротированный refresh).

### 4.5 Провал рефреша — сплит на два класса

- **Транзиентный** (нет сети / timeout / 5xx / Retry-After): токены **сохранить**,
  серый донат + «временная ошибка», ретрай на следующем poll с backoff **60–300 с**
  (как MVP). Валидный грант не выбрасываем из-за просадки сети.
- **Терминальный** (400 `invalid_grant` / отзыв / reuse-detected): грант мёртв →
  `IWidgetTokenStore.Clear`, панель → пустое состояние «Войдите через Anthropic».

В обоих случаях файл Claude Code **не трогаем**. Выбор режима — по §1: в auto с
доступным CC следующий выбор возьмёт borrow; при own-override остаёмся в own с
login-экраном. Ручной «Выход» (§3.5) тоже вызывает `Clear`.

## 5. Доставка и установка (тикет #16)

Деталь:
[Решение: способ доставки и установки](https://github.com/VladPatr96/widget-subscription/issues/16).

- **Форма артефакта**: инсталлятор **Inno Setup** оборачивает **self-contained
  single-file** `WidgetSubscription.exe`
  (`dotnet publish -c Release -p:PublishSingleFile=true`, RID **`win-x64`** —
  рантайм .NET 8 + SkiaSharp внутри). На целевой машине **ничего доустанавливать не
  нужно**. Framework-dependent и bootstrap-загрузка рантайма отклонены.
- **Установка**: exe в `Program Files` (read-only); инсталлятор заводит ярлык и
  деинсталлятор.
- **Автозапуск**: запись `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
  (per-user, без админ-прав), **включена по умолчанию**; в меню трея — **тумблер**
  «Запускать при входе», пишущий/удаляющий ту же запись. Планировщик и
  `shell:startup` отклонены.
- **Только Windows** сейчас. macOS/Linux — **вне области** (§7).
- **Не подписываем**: «Неизвестный издатель» → предупреждение SmartScreen —
  **принятый компромисс** (утилита для своих машин).
- **Каталог данных**: own-токен в `%LOCALAPPDATA%\WidgetSubscription\` (§4.1). При
  **деинсталляции** Inno **спрашивает** «удалить данные и разлогинить виджет?», по
  умолчанию — **оставить** (переустановка не требует повторного логина).

## 6. Иконка приложения (тикет #15)

Деталь:
[Прототип: иконка приложения](https://github.com/VladPatr96/widget-subscription/issues/15),

  (мелкая деталь теряется).

## 7. Граница с Core и сводка seam'ов

Ключевое инвариантное решение (#17 §4): **`Core` не меняется — auth-порта в Core
нет**. Двухрежимность и own-login — специфика провайдера Claude Code, не доменный
концепт. Всё живёт в `Providers.ClaudeCode` рядом с адаптером.

- **`ICredentialSource`** (существующий из MVP §5) остаётся read-seam:
  `Task<AccessToken?> GetAsync(ct)`. Адаптер видит один интерфейс.
- **`SelectingCredentialSource`** — новый композит за `ICredentialSource`: на каждый
  `GetAsync` резолвит активный режим (сохранённый override, иначе авто-детект §1) и
  делегирует borrow- или own-источнику.
  - *borrow-ветка* = существующий `ClaudeCredentialsFileSource` (read-only).
  - *own-ветка* = own-login credential source: реализует `ICredentialSource`,
    внутри — ленивый рефреш (§4.3–4.4), использует `IWidgetLogin` и
    `IWidgetTokenStore`.
- **`IWidgetLogin`** (новый seam, #18): `LoginAsync` = флоу §3.1–3.4, `SignOut` =
  §3.5. Дёргает трей-UI (модальное окно, браузер).
- **`IWidgetTokenStore`** (новый seam, #19): `Load() → WidgetTokens?`,
  `Save(WidgetTokens)`, `Clear()`. Файловая реализация +  in-memory fake для тестов
  (двух-seam test-first дисциплина проекта, как `ICredentialSource`).
- **Персистентный флаг режима** (override §1) хранится внутри композита в
  `Providers.ClaudeCode`.

## 8. Границы эффорта (checklist приёмки)

Спека на передачу считается согласованной, когда зафиксировано:

1. Модель кредов с двумя режимами (borrow read-only / own-login) и авто-выбор с
   персистентным override (§1).
2. Три состояния borrow, протухший токен = деградация (§1).
3. Факты OAuth-флоу: client_id, endpoints, PKCE, ротация refresh, разделение
   грантов, scope (§2).
4. Own-login: loopback + paste-фолбэк, внешний браузер, модальное окно, обработка
   ошибок/таймаута/отмены, повторный вход, «Выход», scope least-privilege (§3).
5. Хранение own-токена: путь/формат, plaintext+ACL, ленивый рефреш+skew,
   single-flight + атомарная запись, сплит транзиент/терминальный (§4).
6. Доставка: Inno Setup поверх self-contained single-file `win-x64`, автозапуск
   `HKCU\...\Run` с тумблером, только Windows, без подписи, каталог данных (§5).
7. Иконка приложения: направление V5 «Spark-in-ring», статичная отдельно от трея
   (§6).
8. Граница с Core неизменна: auth-порта нет; композит + `IWidgetLogin` +
   `IWidgetTokenStore` в `Providers.ClaudeCode` (§7).

Сборка (инсталлятор, publish-скрипт, реализация seam'ов, `.ico`) — **отдельный
эффорт исполнения после подписи**, не входит в эту спеку.

## 9. Вне области (out of scope)

- Реальные интеграции других провайдеров, браузерный скрейп claude.ai, мобильное
  приложение (унаследовано из MVP).
- Несколько аккаунтов/подписок одновременно в одном виджете.
- **Доставка/установка/автозапуск на macOS и Linux** (`.app`/`.dmg` + LaunchAgent;
  `.desktop`/AppImage/`.deb`) — решено только Windows (#16).
- **Подписывание OV/EV Authenticode-сертификатом и раздача третьим лицам** — в #16
  решено не подписывать; SmartScreen — принятый компромисс. Вернётся только при
  redraw назначения (публичная дистрибуция).
